7e-2 - Express, Docker y PostgreSQL

Vamos a ver cómo podemos integrar el uso de PostgreSQL en NodeJS, para no instarl PostgreSQL en el equipo directamente, vamos a utilizar docker.



Instalación de docker y PostgreSQL

Para instalar docker simplemente ejecuta: 

sudo apt install docker.io docker-compose


Crear el contenedor

Enlace a docker hub con la información: https://hub.docker.com/_/postgres


Ejecución: 

docker run --name postgres -e POSTGRES_PASSWORD=password -v ./data:/var/lib/postgresql/data -p 5432:5432 -d postgres

Características:

  • Puerto que utiliza por defecto: 5432/tcp
  • Usuario que ejecuta PostgreSQL: postgres
  • Password: el del POSTGRES_PASSWORD
  • Ruta donde se guardan los datos: /var/lib/postgresql/data


Acceder a un cliente de la terminal

Para acceder al cliente de la terminal: 

docker exec -it -u postgres postgres /bin/bash

Y ejecutamos: psql


También podemos ejecutar la cadena de conexión (esto es útil para probar que está bien construida) 

psql postgres://postgres:password@localhost:5432/productos



Órdenes del cliente


Algunas órdenes:

  • \list o \l → lista las bases de datos disponibles
  • \c catalogo → se conecta a la base de datos catalogo

Como no tenemos creada ninguna base de datos, empezaremos creándola: 

create database catalogo;

Crearemos una tabla producto: 

CREATE TABLE IF NOT EXISTS producto (
id SERIAL PRIMARY KEY,
nombre varchar(40) not null,
precio real
);

E insertaremos un registro: 

insert into producto (nombre, precio) values ( 'Bicicleta', 23.4);

Comprobamos que la inserción fue correcta: 

select * from producto;



Usando un docker-compose y Adminer

Usando un fichero docker-compose.yml podemos crear dos contenedores conectados entre sí que se ejecuten compartiendo la misma red, de manera que podrán comunicarse entre sí. Uno de los contenedores será el de PostgreSQL, el otro el de Adminer.
El Adminer es un cliente web para conectarse y gestionar diferentes bases de datos, entre ellas PostgreSQL.


Fichero docker-compose.yml: 

# Use postgres/example user/password credentials
version: '3.9'

services:

  db:
	image: postgres
	restart: always
	# set shared memory limit when using docker-compose
	shm_size: 128mb
	# or set shared memory limit when deploy via swarm stack
	#volumes:
	#  - type: tmpfs
	#    target: /dev/shm
	#    tmpfs:
	#      size: 134217728 # 128*2^20 bytes = 128Mb
	environment:
	  POSTGRES_PASSWORD: password
	ports:
	  - 5432:5432
	volumes:
	  - ./data:/var/lib/postgresql/data

  adminer:
	image: adminer
	restart: always
	ports:
	  - 8080:8080

El fichero podemos usarlo ejecutando:

  • docker-compose up → crea los contenedores y los ejecuta. Si ya están creados, simplemente los ejecuta.
  • docker-compose down → elimina los contendores que se hubieran creado.

Una vez arrancados, podremos utilizar el Adminer en http://localhost:8080/
Deberemos crear la base de datos igualmente, la tabla producto e insertar algún registro.



Usar PostgreSQL desde nodejs

Vamos a utilizar el paquete pg-promise como cliente para conectarnos y ejecutar sentencias SQL.
https://www.npmjs.com/package/pg-promise
https://github.com/vitaly-t/pg-promise/wiki/Connection-Syntax#connection-string


Instalamos el paquete: 

npm install pg-promise;

Ejemplo de código: 

import pgp from 'pg-promise';

const db = pgp()({
	host: 'localhost',
	port: 5432,
	database: 'productos',
	user: 'postgres',
	password: 'password',
});

db.any('SELECT * FROM producto')
  .then(data => {
	console.log(data);
  })
  .catch(error => {
	console.log(error);
  });

El único aspecto que nos puede resultar extraño es la manera en la que creamos el objeto db. El código es el de abajo: 

const db = pgp()({ ... });

Lo que ocurre es que al ejecutar pgp() se devuelve una función que se ejecuta inmediatamente con el objeto con los datos de la conexión como parámetro. Es decir, es equivalente a esto: 

const funcion_db = pgp();
const db = function_db({
	host: 'localhost',
	port: 5432,
	database: 'productos',
	user: 'postgres',
	password: 'password',
});


Métodos del objeto db

El objeto db tiene varios métodos en función del número de filas que devuelve:

  • none
  • one
  • oneOrNone
  • many
  • manyOrNone, any

Como en la mayoría de clientes de bases de datos, se pueden usar plantillas para evitar el SQL injection. Por ejemplo: 

await db.any('SELECT * FROM product WHERE price BETWEEN $1 AND $2', [1, 10]);

Los datos reales se traspasan mediante un array (o el valor si sólo hay un parámetro en la consulta).


Por ejemplo: 

try{
  const data = await db.one('SELECT * FROM producto where id=$1', 2);
  console.log(data);
}
catch(error){
	console.log(error);
}



Siguientes pasos

Crea un módulo de javascript que al importarlo devuelva el objeto db con la conextión a nuestra base de datos. 


import pgp from 'pg-promise';

const db = pgp()({
	host: 'localhost',
	port: 5432,
	database: 'productos',
	user: 'postgres',
	password: 'password',
});

export default db;
