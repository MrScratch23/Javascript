7c - Formularios y peticiones al servidor

Las peticiones que se realizan desde un formulario con el atributo method pueden ser por defecto de tipo GET y tipo POST.


Para una API tipo REST, sin embargo, se usan otros verbos, por ejemplo supongamos una API de un backend creado en Laravel (o Node JS) que permita gestionar la carta de un restaurante.


La API podría tener estos endpoints: 

Tabla con verbos, URI, Acción y nombre de ruta esperados
+-----------+----------------------------+--------------------------------------------+
| Verbo     | URI                        | Acción                                     |
+===========+============================+============================================+
| GET       | /carta                     | muestra la pantalla de carta               |
+-----------+----------------------------+--------------------------------------------+
| GET       | /carta/createPlato         | muestra la pantalla de crear plato         |
+-----------+----------------------------+--------------------------------------------+
| GET       | /api/plato/{id_plato}      | información de un plato                    |
+-----------+----------------------------+--------------------------------------------+
| POST      | /api/plato                 | crea un plato en la carta                  |
+-----------+----------------------------+--------------------------------------------+
| PUT/PATCH | /api/plato/{id_plato}      | actualiza un plato en la carta             |
+-----------+----------------------------+--------------------------------------------+
| DELETE    | /api/plato/{id_plato}      | elimina un plato de la carta               |
+-----------+----------------------------+--------------------------------------------+

Así, por ejemplo, si recibimos una petición por el método GET a la ruta /carta, se deberá mostrar una pantalla o página web con un listado de todos los platos de la carta y se invocará al método.


Si se recibe una petición a /carta/createPlato usando el método GET, se mostrará una vista un formulario que servirá para crear un nuevo plato en la aplicación.


Si se recibe una petición a /api/plato usando el método POST, se hará una petición para almacenar en la base de datos un plato (creado previamente en el formulario mostrador por el método /carta/createPlato).


Si se recibe una petición a /api/plato/{id_plato} usando el método GET, se pedirá al servidor información detallada de un plato concreto.
Etcétera.



Procesado de formularios y peticiones al servidor con FormData

Hemos visto como usar la API Fetch para realizar peticiones al servidor y recuperar su contenido, también hemos visto un ejemplo de cómo mandar información usando un método POST.


Repetimos aquí el ejemplo: 

fetch("https://dummyjson.com/products/add",{
  method: "POST",
  headers: {
    "Content-type" : "application/json"
  },
  body: JSON.stringify({
    description: "Drones de combate",
    price: "1000",
    rating: "9/10"
  })
})
.then(res=>res.json())
.then(json=>console.log(json))
.catch(error=>console.error(error));

Fíjate que el objeto que mandamos con los datos se estringuifica y se manda en el body de la petición. Este objeto literal lo hemos creado nosotros manualmente.
Sin embargo, en Javascript existe un objeto que representa los datos de un formulario, este es el objeto FormData.


El FormData

Lo habitual es crear un formulario para proceder a mandar la información contenida en el mismo, por ejemplo un formulario de login de un usuario podría mandar el nombre de usuario y la contraseña.


Para mandar los datos al servidor podmos usar un objeto FormData. Por ejemplo, supongamos que tenemos un formulario <form id="frmDatos"> y queremos mandar sus datos al servidor por el método POST y esperar la respuesta que será mostrada en una división.


El código podría ser este: 

const realizarPeticion = async (datos, divResultado) =>{
  try{
    const response = await fetch('servidor.php',{
      method: 'post',
      body: datos
    });
	
    if(!response.ok)
      throw new Error("Error al contactar con el servidor")

    const datos = await response.text();
    divResultado.innerHTML = datos;
  }
  catch(error){
    divResultado.innerHTML = error;
  }
}

document.querySelector("#btnEnviar").addEventListener("click", e=>{
  const frm = document.querySelector("#frmDatos");
  const divResultado = document.querySelector("#divResultado");
  let datosForm = new FormData(frm);
  console.log(datosForm);
  realizarPeticion(datosForm, divResultado);	
});

Fíjate que al crear el objeto FormData se le ha pasado el objeto del formulario. El objeto generado contendrá en pares clave-valor todo el contenido de todos los controles del formulario que tengan un atributo name.
Los controles sin atributo name serán ignorados.


El objeto FormData se puede crear vacío en lugar de partir de los datos de un formulario y en cualquier caso siempre se puede añadir más datos o quitarlos usando los siguientes métodos:

  • append( "clave", valor);
  • delete("clave");

Funciones de ayuda

Veamos un par de funciones de ayuda.


Petición de tipo GET

/**
 * Realiza peticiones AJAX de tipo GET
 * @param {string} url 
 * @param {FormData} parametros - Objeto FormData con los parámetros de la llamada 
 * @param {Boolean} json - Indica si la respuesta debe ser un fichero JSON o de texto
 * @returns 
 */
const peticionGET = async (url, parametros, json=true) => {
 
  let searchparam = new URLSearchParams( parametros );
  let urlget = url + "?" + searchparam.toString();

  let response = await fetch( urlget);
   
  if(!response.ok) {
    console.error("Error al acceder al acceder al servidor (STATUS != 200..299).");
    throw new Error("Error al acceder al acceder al servidor (STATUS != 200..299).");
  }
 
  if(json)
    return await response.json();
  else
    return await response.text();
}

Un ejemplo de uso podría ser este, imagina que tienes un formulario en el que se puede elegir el id de un producto y queremos que el servidor nos responda con los datos de ese producto. El servidor responderá con texto. 

const getProducto = async () =>{
  const frmProducto = document.querySelector("#frmProducto");
  const formData = new FormData(frmProducto);

  try{
	   const textoRespuesta = await peticionGET("/php/getProducto.php", formData, false);
    document.querySelector("#pRespuesta").innerHTML = textoRespuesta;
  }
  catch(error){
    document.querySelector("#pRespuesta").innerHTML = error;
  }
}


Petición de tipo POST

/**
 * Realiza peticiones AJAX de tipo POST
 * @param {string} url 
 * @param {FormData} parametros - Objeto FormData con los parámetros de la llamada 
 * @param {Boolean} json - Indica si la respuesta debe ser un fichero JSON o de texto
 * @returns 
 */
const peticionPOST = async (url, parametros, json=true) => {

  let response = await fetch( url,{
    body: parametros,
    method: "POST"
  });
   
  if(!response.ok) {
    console.error("Error al acceder al acceder al servidor (STATUS != 200..299).");
    throw new Error("Error al acceder al acceder al servidor (STATUS != 200..299).");
  }
 
  if(json)
    return await response.json();
  else
    return await response.text();
}

Su uso es similar al de la petición GET.



Importante

En esos ejemplos, se mandan los datos contenidos en el objeto de tipo FormData directamente al servidor. Eso es cómodo y conveniente, pero hay que tener en cuenta que el se mandarán en una petición con el tipo de contenido multipart/form-data.
Si el servidor lo puede procesar directamente como ocurre con PHP no tendremos problemas, pero es posible que con Nodejs sí tengamos que tenerlo en cuenta u utilizar un paquete de middleware para procesarlo como multer.


Si necesitamos convertir un objeto FormData a JSON, Hay varias maneras de hacerlo: 

function formDataToJson(formData) {
  return JSON.stringify( 
    Object.fromEntries(
      Array.from(formData.keys()).map(key => [
	    key, formData.getAll(key).length > 1 ? 
	      formData.getAll(key) : formData.get(key)
      ])
    )
  )
}

Otra manera sería esta: 

function formDataToJson(formData) {
	const obj = {};
	formData.forEach((value, key) => {
		// Si ya existe una clave, la convertimos en un array
		if (obj[key]) {
			if (Array.isArray(obj[key])) {
				obj[key].push(value);
			} else {
				obj[key] = [obj[key], value];
			}
		} else {
			obj[key] = value;
		}
	});
	return JSON.stringify(obj);
}

Habrá que adaptar la función de peticiónPOST para que indique que se está mandando un JSON: 

/**
 * Realiza peticiones AJAX de tipo POST
 * @param {string} url 
 * @param {FormData} parametros - Objeto FormData con los parámetros de la llamada 
 * @param {Boolean} json - Indica si la respuesta debe ser un fichero JSON o de texto
 * @returns 
 */
const peticionPOST = async (url, parametros, json = true) => {
  // Convertir FormData a JSON
  const jsonData = formDataToJson(parametros);

  let response = await fetch(url, {
    body: jsonData,
    method: "POST",
    headers: {
      'Content-Type': 'application/json'
    }
  });

  if (!response.ok) {
    console.error("Error al acceder al servidor (STATUS != 200..299).");
    throw new Error("Error al acceder al servidor (STATUS != 200..299).");
  }

  if (json)
    return await response.json();
  else
    return await response.text();
}
