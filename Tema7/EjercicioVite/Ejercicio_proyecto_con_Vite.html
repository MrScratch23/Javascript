Ejercicio proyecto con Vite

Vite es un bundler o empaquetador de código, además de ser un servidor de desarrollo y tener soporte para test unitarios. Suele utilizarse mucho para crear la parte del cliente de una aplicación web.


Crea una carpeta con el nombre proyectovite y abre una terminal en esa carpeta. 

npm create vite@latest .

En las opciones que te aparence selecciona:

  • Vanilla (sin framework)
  • Javascript
  • No queremos usar rolldown-vite
  • No queremos que instale dependencias y comience

Cuando haya terminado, abre el Visual Studio Code en esa carpeta y mira el fichero package.json. Ahí aparece parte de la documentación del proyecto, como su nombre, la versión, los scripts y las dependencias.


Vuelve a la terminal y verás que al crearse el proyecto, lo último que nos decía era qué teníamos que ejecutar. Hazlo ahora: 

npm install
npm run dev

La primera orden nos descargará los paquetes que necesita Vite, la segunda nos arrancará el script de desarrollo.
Por defecto nos ha creado un servidor en el puerto 5173. Ve al navegador y ábrelo.


Cuando hayas terminado de toquetear la web, vuelve al VSCode y elimina todo lo que haya en la carpeta src. Crea en su interior las carpetas src/js y src/css.


Ahora modifica el fichero index.html de la raíz para que sea este: 

<!doctype html>
<html lang="es">
  <head>
    <meta charset="UTF-8" />
    <link rel="icon" type="image/svg+xml" href="/vite.svg" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>borrar</title>
    <script type="module" src="/src/js/index.js"></script>
    <link rel="stylesheet" href="/src/css/simple.css">
  </head>
  <body>
    <h1>Hola caracola</h1>
    <p>Un proyecto interesante.</p>
    <form>
      <button type="button" id="btnAcción">Acción</button>
    </form>
  </body>
</html>

Como verás, queremos usar SimpleCSS, pero no desde la web, sino servirlo desde nuestro propio servidor, así no dependeremos de nada externo.
Vuelve a la terminal, para Vite usando la combinación de teclas Ctrl + C y ve a la carpeta src/css. Ahí ejecuta lo siguiente: 

wget https://cdn.simplecss.org/simple.css

Eso descargará el SimpleCSS sin minificar.


Vuelve a arrancar Vite (npm run dev) y mira que la web funcione como se espera. En VSCode podrás ver que tenemos el fichero simple.css sin minificar.


Crea el fichero src/js/index.js con este contenido: 

//import { annotate } from 'rough-notation';
// Or using unpkg
import { annotate } from 'https://unpkg.com/rough-notation?module';

const e = document.querySelector('#btnAcción');
const annotation = annotate(e, { type: 'underline' });
annotation.show();

Fíjate que ahora al iniciar la web se subraya el botón. Estupendo, eso es que estamos usando la bilioteca Rough-Notation (https://roughnotation.com/). Sin embargo, si te fijas lo está cargando desde un servidor externo. Luego lo arreglaremos, ahora vamos a ver otra cosa primero.


Vuelve a la terminal y para vite, vamos a construir nuestra aplicación cliente. Desde la terminal si ejecutas npm run podremos ver los scripts que hay disponibles. Antes ejecutamos npm run dev para arrancar Vite en modo desarrollo, ahora ejecuta el de construcción: 

npm run build

Eso nos ha generado un proyecto web minificado en la carpeta dist (que acaba de ser creada). Ese dist viene de distribution y es el proyecto finalizado, listo para ser desplegado (o distribuido).
Fíjate que el fichero index.html ha sido movido y se ha creado una carpeta assets que contiene un fichero JS y uno CSS, ambos minificados. También ha copiado el fichero vite.svg que está en la carpeta public.
Eso es lo que hace Vite al construir el proyecto: minificar y procesar todos los archivos e imágenes que haya en src, pero deja intactos los que estén en public.


Vamos a cargar Rough-Notation desde nuestro servidor. para ello volvemos a la raíz de nuestro proyecto en la terminal y ejecutamos: 

npm install rough-notation

Fíajte que ahora se ha añadido a nuestro package.json como una dependencia. Vite aparece como una dependencia también, pero sólo durante el desarrollo.


Modifica el fichero src/js/index.js con este contenido: 

import { annotate } from 'rough-notation';
// Or using unpkg
//import { annotate } from 'https://unpkg.com/rough-notation?module';

const e = document.querySelector('#btnAcción');
const annotation = annotate(e, { type: 'underline' });
annotation.show();

Comprueba que todo sigue funcionando correctamente cuando arrancas Vite (de nuevo npm run dev).
Vuelve a construir el proyecto, para Vite y ejecuta npm run build.
Comprueba ahora el fichero javascript que está en dist/assets, es imposible saber que había ahí: se ha minificado tanto nuestro código como el de la biblioteca en un único fichero, así cargará antes al realizarse una única llamda al servidor.


Perfecto, pero queremos cargar la biblioteca sólo cuando se pulse el botón de Acción, así cambia el fichero src/js/index.js con este contenido: 

const btnAccion = document.querySelector('#btnAcción');

btnAccion.addEventListener("click", async e=>{
  const {annotate} = await import ('rough-notation');
  
  const annotation = annotate(btnAccion, { type: 'underline' });
  annotation.show();
})

Comprueba que todo funciona correctamente en modo desarrollo y, de nuevo, construye el proyecto final para ver el cambio. No te indico otra vez las órdenes necesarias, ya lo he hecho un par de veces, búscalas arriba.


Fíjate que ahora en dist/assets tenemos dos ficheros javascript: uno que corresponde a la biblioteca y otro que corresponde a nuestro código. Intenta buscar el btnAcción en él y verás que nosotros llamamos a esa variable btnAccion, pero al minificarse se ha cambiado a una única letra para ahorrar unos pocos bytes.


Finalmente arranca un nuevo VSCode en la carpeta dist y usa el liveServer para comprobar que efectivamente el proyecto es el mismo, salvo que optimizado.
