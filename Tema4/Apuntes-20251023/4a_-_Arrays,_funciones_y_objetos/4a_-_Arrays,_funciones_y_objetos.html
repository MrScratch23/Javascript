4a - Arrays, funciones y objetos

Las funciones anónimas y las funciones flecha son dos elementos fundamentales en javascript que se utilizan muchísimo en programación funcional. Las funciones pueden entenderse como trozos de código con nombre que se usan para reutilizar el código y para mejorar la legibilidad del mismo.



Funciones

Las funciones en JS se pueden definir de varias maneras, la más sencilla es esta: 

function nombreFuncion( param1, param2, param3 ){
  /* código */
  return valor;
}

También usando una función anónima: 

const nombreFuncion = function ( param1, param2, param3 ){
  /* código */
  return valor;
}

Y finalmente usando una función flecha: 

const nombreFuncion = ( param1, param2, param3 ) => {
  /* código */
  return valor;
}

Funciones flecha

Partimos de una función clásica: 

function cuadrado( num ) { return num*num; }

Se puede reescribir como una función flecha. 

const cuadrado = (num) => { return num*num; }

Como sólo tiene un único argumento, se pueden quitar esos paréntesis: 

const cuadrado = num => { return num*num; }

Como además sólo hay una instrucción que es un return, se puede quitar el return y las llaves. 

const cuadrado = num => num*num;


Paso de parámetros

Los parámetros que recibe una función son por valor en casos simples (números, booleanos,...) y por referencia en el caso de objetos y arrays.


Cuerpo

  • Las variables creadas dentro de la función son locales a esa función.
  • Una función puede llamarse a sí misma para usar recursividad.
  • Una función puede devolver otra función.
  • Es posible salir antes de una función usando un return;
  • Dentro de una función pueden crearse otras funciones locales que además pueden acceder a las variables de la función padre (esto se llama closure). Esto se considera una técnica avanzada de Javascript, pero en realidad no es tan extraordinario el usarla.
function multiplica(x){
  function interior(y) {
    return x * y;
  }
  return interior; //devolvemos la función llamada interior
}

let doble     = multiplica(2);
let quintuple = multiplica(5);
console.log( doble(10) );     //20
console.log( quintuple(10) ); //50

  • Una función puede recibir cualquier número de argumentos. Si queremos poder procesarlos todos se puede usar arguments (que se comporta como un array de parámetros, es iterable y tiene una propiedad length), aunque es mejor usar un el operador ... (básicamente el resto) que los mete en un array.
function multiplica(multiplicador, ...resto) {
  return resto.map((x) => multiplicador * x);
}
multiplica(2,3,4,5); // [6, 8, 10] (multiplica el primer valor por los restantes creando un array

//Otro ejemplo: una función que devuelve el número de argumentos que recibe
const numArgs = (...args) => args.length;

  • Podemos indicar parámetros predeterminados (siempre serán los últimos)
function multiplicar(a, b=2) {
  return a * b;
}
multiplicar(5);    //10
multiplicar(5, 3); //15

function mult3(a, b=2, c=3) {
  return a * b *c;
}
mult3( 1 );       // 6
mult3( 2, 5)      // 30
mult3( 2, 5, 4 ); // 40

Nota: las funciones creadas con function sufren hoisting, que significa que se mueven virtualmente arriba del código al ser parseado por el navegador. De esta manera se pueden usar funciones en el código que son declaradas más abajo en el código. Esto no ocurre con las funciones anónimas que se asignan con const, let o var, por lo que se recomienda tener la costumbre de declarar las funciones arriba antes de usarlas.
